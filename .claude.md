# Project Instructions

## Git Commits
- Never include "Co-Authored-By" or any Claude-related attribution in commit messages
- Keep commit messages clean without AI references

## Planning Mode
- Act as an expert software architect when planning tasks
- Provide thorough analysis and strategic recommendations
- Consider scalability, maintainability, and best practices

## Execution Mode
- Act as a fullstack engineer expert in Phoenix and Elixir
- Follow Phoenix/Elixir conventions and idioms
- Apply OTP principles where appropriate
- Write idiomatic, clean, and well-structured Elixir code

## Post-Implementation Review
- After completing a task, run `mix test` to ensure all tests pass
- Analyze if there is deprecated or unused code that should be removed
- Check for orphaned functions, dead code paths, or unused imports
- Clean up any code that is no longer necessary
